% Script #1
% Guide to loading files for Approach-Avoidance Task
% Updated November 27, 2024 (Humza NZ)

After running the code, important to check:
	- pre_table.Summary
	- In particular, ensure that trial list lines up with reality (TrialValueType)
	- Check that the time delays for A vs B synchronization are reasonable
	- Check that block assignments are correct (A_BlockIndex, A_trialinfo)
	- Check any Command Window warnings
	- Look at all figures 
    - Check pre_table.trial_RMS and pre_table.trial_R
	- Make sure output is saved appropriately

Things to have before running Matlab code:

A folder with:

- .csv output files for headset A and B, for each Unity recording 
- Motive Optitrack files
- .mat manually generated files for TrialExclude,TrialValueType,TClocations

Input in beginning of TaskFileLoad.m:

- folder (Location of files)
- numUnityFile (How many unity files?)
- UnityMarks (Do the unity files have a log for marks, 1 = Yes, 0 = No/Useless)
- UnityA{n}/UnityB{n} (csv file names for headset A/B log, file #n)
- LinetoExclude_A/B (Ignore a particular message line in the Unity file, used when task is ended mid-block)
- MotiveOptiExist (Does Optitrack data from Motive exist, 1 = Yes)
- MBreaks_Unity
	- Which trials in Unity correspond with new Motive recordings?
	- Use whole number to exclude entire trial (e.g., 72)
	- Use fraction/.5 to not exclude any trial (e.g., 60.5 - only use between blocks)
	- Format: {[a1 a2];[b1 b2 b3]}; where a1-2 are breaks in Unity File 1, b1-3 are breaks in Unity File 2, etc.
- UBreaks
	- Which trials in Unity correspond with new Unity recordings?
	- Always use fraction/.5 (e.g., 40.5)
- Motive{n} (csv file names for Motive file #n)
	- Note: # of motive files must match # of rows in MBreaks_Unity + # of values across rows in MBreaks_Unity
- MotiveARot (motive file columns corresponding to yaw/pitch/roll of participant)
	- n x 3 matrix where each row is a different motive file, in order
- MotiveAPos (motive file columns corresponding to x/y/z position of participant)
	- n x 3 matrix where each row is a different motive file, in order
- MotiveBRot (motive file columns corresponding to yaw/pitch/roll of experimenter)
	- n x 3 matrix where each row is a different motive file, in order
- MotiveBPos (motive file columns corresponding to x/y/z position of experimenter)
	- n x 3 matrix where each row is a different motive file, in order
- A_UseOptiFromMotive (Do we use the Participant OptiTrack data from Motive, 1 = Yes)
	- n x 1 matrix where each row is a different motive file, in order
- B_UseOptiFromMotive (Do we use the Experimenter OptiTrack data from Motive, 1 = Yes)
	- n x 1 matrix where each row is a different motive file, in order
- Opti2Unity_translate (Translation of center points from OptiTrack to Unity)
	- Written down x/y/z values on Motive when rigid body is constructed at the Unity (0,0,0) point
	- n x 3 matrix where each row is a different motive file, in order
	- Separate variables for A and B
- Opti2Unity_direction (Do position axes need to be flipped, -1 if Yes, 1 if No)
	- n x 3 matrix where each row is a different motive file, in order
- OptiRot_offsetA/OptiRot_offsetB (adjustment offsets for the yaw/pitch/roll for Motive)
- OptiRot_direction (Are yaw/pitch/roll in the opposite direction, -1 if Yes, 1 if No)
	- Assumes that straight ahead (Unity +z) is yaw=0, turning right increases yaw (up to 180): range -180 to 180
	- Assumes that looking up increases pitch
	- n x 3 matrix where each row is a different motive file, in order
- interp_column (for correlating ML2/Motive positions to obtain synchronization)
	- 1 for x axis, 2 for y axis (not recommended), 3 for z axis
- horizontalFOV/verticalFOV (field of view in the AR headsets - conservative estimate predetermined through in-person testing)
- AutoSave (to save results and plots in a .mat file, 1 if Yes, 0 if No)
- UseHeadA (is rotation data, either OptiTrack or Magic Leap 2,  for headset A usable for analysis, 1 if Yes, 0 if No)
- UseEye (is eye movement data from headset A usable for analysis, 1 if Yes, 0 if No)
- UseBiopac (are we using biopac, 1 if Yes, 0 if No)
- UseRNS (are we using RNS, 1 if Yes, 0 if No)

Variables (.mat) in the folder/workspace:

- TrialValueType variable
	- 4 columns, each row corresponds to a trial
	- Column 1: Block # (0-7)
	- Column 2: Reward
	- Column 3: Penalty
	- Column 4: Trial Type (optional to start, assigned in the subsequent code anyway)
		- 1: 100/1000, 2: 500/500, 3:1000/100, 4:500/0
- TClocations variable (treasure chest locations)
	- 3 columns, must have exact correspondence to TrialValueType
	- Column 1: Left/Right dim (of logo)
	- Column 2: Height dim (should all be 0)
	- Column 3: Forward/Backward dim (of logo)
- TrialExclude variable
	- List of trials (corresponding to TrialValueType) to exclude
	- Can be empty (TrialExclude=[];)

=============================================

"TaskFileLoad.m" script takes these raw files (with manually determined info obtained from task notes) to import and arrange data for manipulation
Imported data is kept in "pre_table" variable

Subscripts:
Timestamp - pre_table.A_timestamp, pre_table.B_timestamp
	- imports timestamp as hours:minutes:etc

AppTime - pre_table.A_AppTime, pre_table.B_AppTime
	- timestamp converted into seconds

GlobalBlock - pre_table.A_GlobalBlock, pre_table.B_GlobalBlock 
	- an internal indicator for task stages (not used)

LogType - pre_table.A_LogType, pre_table.B_LogType
	- indicates if row is a real datapoint (1) or an event marker (0). For Unity output, event markers and data are presented in series rather than parallel
	- Output is sometimes "2" which should represent the 'Log restarted to resume task' message after successful coin collection

EyesOpen -  pre_table.A_EyesOpen, pre_table.B_EyesOpen
	- indicates if both eyes are open (2) or one eye is open (1). A score of 0 means either eyes are closed or an event is being logged at that row

EyeTarget - pre_table.A_EyeTarget, pre_table.B_EyeTarget
	- Where are participants looking?
	- WARNING: has not been tested (!)
	- Is probably all wrong, I don't use this
	- 1: Other Participant
	- 2: Closed lock
	- 3: Game Controller
	- 4: Ceiling
	- 5: Ground
	- 6: East
	- 7: North
	- 8: South
	- 9: West
	- 10: None, 11: Nothing

Unity Head Position (Anchored) - pre_table.UA_HeadPos, pre_table.UB_HeadPos
	- same dimensions as the treasure chest file

Unity Head Forth (Anchored) - pre_table.UA_HeadFrth, pre_table.UB_HeadFrth
	- Head direction: 3D projections on x,y,z - can be converted to yaw/pitch

Unity Eye Direction (Anchored) - pre_table.UA_EyeDir, pre_table.UB_EyeDir
	- Gaze: 3D projections on x,y,z - can be converted to yaw/pitch

Unity Fixation Point (Anchored) - pre_table.UA_FixPt, pre_table.UB_FixPt
	- theoretically, where the participant is looking
	- PROBABLY WRONG, I suggest to ignore this parameter

Actual Used Head Position/Direction - pre_table.A_HeadPos/HeadDir/HeadFrth, pre_table.B_HeadPos/HeadDir/HeadFrth
	- derived from either unity or calculated/aligned from Motive
	- depends on code settings (MotiveOptiExist, A_UseOptiFromMotive, B_UseOptiFromMotive) 
	- HeadDir: yaw/pitch (and sometimes roll, if data was derived from Motive)
	- HeadFrth: 3D projections on x,y,z 

Task Messages from Headset A - pre_table.A_MCode (derived from pre_table.A_Message)
	- 1: start of experiment
	- 2: end of intermission/start of trial
	- 3: successful chest approach
	- 4: penalty/collision :(
	- 5: time expired 
	- 6: coin collection (after chest approach)
	- 7: log restart after coin collection
	- 8: end of block
	- 20: ID of upcoming block (may not be provided consistently)
	- 21: upcoming chest location (may not be provided consistently)
    	- 24: Automatic Mark sent (just for inter-headset calibration)
	- 25: Manual Mark sent (for RPi/Biopac/RNS)
	- 26: start of a block
	- 999: unknown log file message

Task Messages from Headset B - pre_table.B_MCode (derived from pre_table.B_Message)
	- 11: start of entire task
	- 12: start of each block
	- 13: coin collected
	- 14: time expired
	- 15: collision
	- 16: penalty
	- 17: log restart after coin collection
	- 18: finished "preventing" in block - basically, block end
	- 19: block end
	- 20: ID of upcoming block (may not be provided consistently)
	- 21: upcoming chest location (may not be provided consistently)
        - 24: Automatic Mark sent (just for inter-headset calibration)
	- 25: Manual Mark sent (for RPi/Biopac/RNS)
	- 999: unknown log file message

Head Velocity - pre_table.A_HeadVel, pre_table.B_HeadVel
	- velocity of both individuals in two dimensions (height ignored)

Block Index - pre_table.A_BlockIndex
Trial Index - pre_table.A_TrialIndex

Trial Info - pre_table.A_trialinfo
	- Matrix of all trials in the file
	- Each row is a trial
	- Column 1-2: index of trial start/end
	- Column 3-6: TrialValueType
	- Column 7: Preceding intermission index
	- Column 8: Subsequent collection index (if successful)
	- Column 9: Successful collection? 1 if Yes, 0 if No
	- Column 10: How many collisions?
	- Column 11: Synchronization offset from Headset A to B (seconds)
		- calculated from "collection" markers in each log
	- Column 12: Synchronization offset from Headset A to B (seconds)
		- calculated from marks, and modulated within each block by "collection" markers
	- Column 13: Detour distance
	- Column 14: Average speed of participant (m/s)
	- Column 15: Average speed of experimenter (m/s)
	- Column 16: Average distance between participant and experimenter (m)
	- Column 17: Angle of avoidance between participant and experimenter (deg.)
	- Column 18: Average distance between participant and treasure chest (m)
	- Column 19: Average distance between experimenter and treasure chest (m)

Time Difference - pre_table.A_TimeDiff
	- Passage of time from each log to the next

Position of B, sychronized to A using "collection" logs - pre_table.UB_SynchPos
B head orientation, synchronized to A by "collection" logs - pre_table.UB_SynchFrth

Unity trial #s with new Motive recordings - MBreaks_Unity

Angle between experimenter's heading direction and participant location - pre_table.AvoidAngle
Distance between participant and treasure chest - pre_table.P1_Dist_to_Chest
Distance between participant and experimenter - pre_table.P1_Dist_to_P2
Distance between experimenter and treasure chest - pre_table.P2_Dist_to_Chest

Treasure chest in view (binary) - pre_table.TCinView
Experimenter in view (binary) - pre_table.P2inView

Direction of treasure chest (top part) relative to participant head direction (yaw/pitch) - pre_table.TCDir
Direction of experimenter (midsection) relative to participant head direction (yaw/pitch) - pre_table.P2Dir
Direction of treasure chest (top part) relative to participant gaze (yaw/pitch) - pre_table.TC_Gaze
Direction of experimenter (midsection) relative to participant gaze (yaw/pitch) - pre_table.P2_Gaze
	- For these four, (+) yaw is to the right of the participant head/gaze, and (+) pitch is above

Participant gaze (yaw/pitch) - pre_table.GazeDir
2D eye position coordinates - pre_table.Derived_EyeXY
	- Derived from difference between 3D Gaze and Head Orientation vectors
	- converted to yaw/pitch
2D eye movement speed - pre_table.Derived_EyeVel
	- Units: degrees/sec

Behavioral Index - pre_table.BehaviorIndex (v1 - v4)
Proportions of each behavior for all trials - pre_table.BehaviorSummaries (v1-v4)
	
v1 - 1: No Conflict, 2: Passive Avoidance, 3: Active Avoidance, 4: Conflicted Pursuit
v2 - 1: No Movement, 2: Approach, 3: Avoidance, 4: No Conflict, 5: Sabotage
v3 - Same as v2, but relation with experimenter is based on angles rather than distance
v4 - Very strict criteria for approach, avoidance, and no conflict
	- 1: Approach (Speed > 0.2 m/s, getting closer to chest, getting closer to experimenter, decreasing angle of avoidance)
	- 2: Avoidance (Speed > 0.2 m/s, getting further from chest, getting further from experimenter, increasing angle of avoidance)
	- 3: No Conflict (Speed > 0.2 m/s, getting closer to chest, getting further from experimenter, increasing angle of avoidance)
	- 4: Other (all other data points)

Behavior Summaries - pre_table.BehaviorSummaries.v[1-4]
	- Proportion of each trial occupied by each behavior
	- Done separately for each behavior coding system
	- v2 is the system that is actually subsequently used

Mark_AB_diff - Synchronization offset from Headset A to B (seconds)
	- calculated from marks in each log

Indexes of noisy eye movement timepoints - pre_table.BadEyeIdx
	- physiologically impossible/unlikely eye positions/velocities

Summary: Summary of information to be checked
	- Total # of blocks
	- Total # of blocks analyzed
	- Total # of trials
	- Total # of trials analyzed
	- List of analyzed trials
	- Analyzed trials type 1
	- Analyzed trials type 2
	- Analyzed trials type 3
	- Analyzed trials type 4
	- Total # of collections
	- Total # of collections analyzed
	- List of synchronization offsets (marks)
	- List of synchronization offsets (collections)
	- % of trials with collision
	- % of trials with target reached
	- Discrepancy between treasure chest and trial end location (should be ~0.5m)
	- Average participant speed (m/s)
	- Average experimenter speed (m/s)
	- Distance between participant/experimenter during collisions (m)
	- Total amount of data during good trials (seconds, data points)
	- % of indexes in good trials with noise eye movement data
	- % of indexes in good trials with treasure chest in view
	- % of indexes in good trials with experimenter in view
	- Unity index corresponding to Motive files - each row is a Unity/Motive match
	- R value correspondence between Unity/Motive - each row is a Unity/Motive match
	- RMS value correspondence between Unity/Motive - each row is a Unity/Motive match
